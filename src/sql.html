DATATYPES AND CONSTRAINTS IN SQL

DATATYPES :-

DATATYPE DATATYPE--INFO TYPE
INT USED TO STORE INTEGER VALUES NUMERICAL
DECIMAL(n,d) USED TO STORE DECIMAL VALUES.
n MEANS NUMBER OF DIGITS BEFORE DECIMAL AND d MEANS NUMBER OF DIGITS AFTER DECIMAL		 NUMERICAL
CHAR(maxSize) USED TO STORE ALPHANUMERIC OR OTHER SYMBOLS TOO. maxSize IS GIVEN TO ALLOW MAX INSERTION OF THE CHARACTERS WHICH CAN TAKE PLACE TEXT
VARCHAR(maxSize) USED TO STORE ALPHANUMERIC OR OTHER SYMBOLS TOO. maxSize IS GIVEN TO ALLOW MAX INSERTION OF THE CHARACTERS WHICH CAN TAKE PLACE TEXT
ENUM(value1,value2,...valueN) ALLOWS STORAGE OF THE GIVEN VALUES ONLY THAT IS value1,value2,.....valueN TEXT
DATE ALLOWS THE STORAGE OF DATE IN "yyyy/mm/dd" WHERE "/" CAN BE REPLACED BY HYPHEN "-" DATE/TIME
TIME ALLOWS THE STORAGE OF TIME IN THE FORMAT hh:mm:ss DATE/TIME
YEAR(2 or 4) ALLOWS THE STORAGE OF YEAR ONLY . YEAR(2) ALLOWS THE STORAGE OF ONLY LAST TWO DIGITS WHILE YEAR(4) ALLOWS STORAGE OF 4 DIGITS OF THE YEAR DATE/TIME
DATETIME ALLOWS THE STORAGE OF DATE FOLLOWED BY TIME IN THE GIVEN FORMAT yyyy-mm-dd hh-mm-ss OR yyyy/mm/dd hh:mm:ss DATE/TIME

CONSTRAINTS :-

CONSTRAINT SYNTAX
PRIMARY KEY 1. AT COLUMN LEVEL :-
colName DATATYPE PRIMARY KEY

2. AT ROW LEVEL
PRIMARY KEY ( nameOfCol1 , nameOfCol2,.... )
NOTE THAT COMPOSITE PRIMARY KEY CAN BE APPLIED AT ROW LEVEL ONLY
UNIQUE KEY 1. AT COLUMN LEVEL :-
colName DATATYPE UNIQUE KEY

2. AT ROW LEVEL
UNIQUE KEY ( nameOfCol1 , nameOfCol2,.... )
NOT NULL nameOfCol DATATYPE NOT NULL
DEFAULT nameOfCol DATATYPE DEFAULT value

OPERATORS IN SQL

RELATIONAL OPERATORS

GENERAL SYNTAX :-
OPERAND1 OPERATOR OPERAND2
RESULT :- true or false
OPERATOR EXAMPLE
> age>10
>= date>="2004/07/13"
< salary<150000
<= rollNo<=5
= name="VIKAS"
!= OR <> age!=15
OR
age<>15

BETWEEN OPERATOR

BETWEEN value1 AND value2
VALUE1 AND VALUE2 ARE INCLUDED
EXAMPLE :-
age BETWEEN 10 AND 15
NOT BETWEEN IS USED TO FIND RESULT WHICH IS NOT IN BETWEEN THE SPECIFIED VALUES
age NOT BETWEEN value1 AND value2

IN OPERATOR

IN(value1,value2,....)
EXAMPLE :-
salary IN(10000,20000,30000)
NOT IN IS USED TO FIND THE RESULT HAVING VALUES NOT IN SPECIFIED VALUES
salary NOT IN(10000,20000,30000)

AND OPERATOR

USED TO COMBINE TWO CONDITIONS AND GIVES RESULT TRUE ONLY WHEN ALL CONDITIONS ARE TRUE
operand1 AND operand1
OR
operand1 && operand1
NOTE THAT "AND" USED IN BETWEEN OPERATOR IS NOT AND OPERATOR

OR OPERATOR

USED TO COMBINE TWO CONDITIONS AND GIVES RESULT TRUE ONLY WHEN ANY ONE CONDITION IS TRUE
operand1 OR operand1
OR
operand1 || operand1

NOT OPERATOR

GIVES RESULT TRUE IF CONDITION IS FALSE AND VICE-VERSA
NOT operand/condition
OR
! operand/condition
NOTE THAT "NOT" USED IN "NOT BETWEEN" AND "NOT IN" IS NOT "NOT" OPERATOR
EXAMPLE :-
NOT salary BETWEEN 10000 AND 300000
