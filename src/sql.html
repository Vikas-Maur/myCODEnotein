DATATYPES

NUMERICAL --> INT ,
DECIMAL(n,d)
n represents no. of digits before decimal
and d represents after decimal
TEXT --> VARCHAR(maxSize) ,
CHAR(maxSize) ,
ENUM (value1,value2,...)
DATE/TIME --> DATE
TIME
YEAR(2 OR 4)
DATETIME
DATE IS STORED IN THE FORMAT :- yyyy/mm/dd OR yyyy-mm-dd
TIME IS STORED IN THE FORMAT :- hh:mm:ss
YEAR(2) --> stores last two digits of year
YEAR(4) --> stores the full year

CONSTRAINTS

1.PRIMARY KEY
CAN BE APPLIED AT ROW/COLUMN LEVEL

SYNTAX FOR COLUMN LEVEL
colName DATATYPE PRIMARY KEY

EXAMPLE FOR ROW LEVEL :-
CREATE TABLE student VALUES (sno INT , age INT , name CHAR(30), PRIMARY KEY(colName1,colName2,...));
NOTE THAT COMPOSITE PRIMARY KEY CAN BE APPLIED AT ROW LEVEL ONLY
2.UNIQUE KEY
CAN BE APPLIED AT ROW/COLUMN LEVEL
SYNTAX IS SAME AS DESCRIBED IN PRIMARY KEY
3. NOT NULL
4. DEFAULT value
THESE TWO CONSTRAINTS CAN BE APPLIED AT COLUMN LEVEL ONLY

QUERYING IN TABLES

TO SEE RECORDS

SELECT colName1,colName2,... FROM nameOfTable ;
NOTE THAT * REPRESENTS ALL COLUMNS

FILTERING THROUGH RECORDS

WITH THE HELP OF SELECT COMMAND AND USE OF
WHERE CLAUSE AT THE END OF SELECT COMMAND STATEMENT RECORDS CAN BE FILTERED
SELECT colName1,colName2,... FROM nameOfTable
WHERE condition ;
