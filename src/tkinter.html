Methods Of Tk Class / Configuring root

Setting Up Root Title

Setting Up root geometry

Making The window transparent

root.attributes("-alpha",value)
# value lies between 0 (fully-transparent) and 1 (full-non-transparent)

Making the window remain at top of all windows

    CODE
    DESC
root.attributes("-topmost",value)
value is either --> 1 or 0
1 signifies that window must be at the top of other windows (always)
0 signifies that window may or may not be at the top of other windows

Making the GUI full screen

    CODE
    DESC
root.attributes("-fullscreen",value)
Value=0 represents the window is not fullscreen.
Value>0 represents full screen mode

Grid , Pack and Config methods

Pack

syntax: widget.pack(**opt)
**opt includes :
side: LEFT,RIGHT,TOP,BOTTOM
expand: True,False
fill: X , Y , BOTH
For ex: LEFT is a variable made in tkinter which represents "left" Similarly other "side" and "fill" values works

Grid

syntax: widget.grid( row= rowNo , column=columnNo ,**opt)
**opt includes:
sticky: "n","s","e","w","nsew" , ..etc
To know other values you can put wrong values in it. Then tkinter will throw an error which will tell you the correct values

Note that:

Pack and Grid cannot be used with each other
For ex : Suppose a widget is packed in root . Then another widget cannot be gridded in root.
For That purpose frames are used which behave like root for its inner widgets

    DESC

Events in tkinter

syntax: widget.bind( "<Event>",func=function)
# Note that tkinter passes an argument to the given function
# Run dir(event)
where event is the argument given to function

The Function is something like

def function(event):
dir(event)
# event argument is given by tkinter

Use Of Lambda Function

# Used to call argumented functions
widget.bind("<Event>",
,func= lambda event: funcToCall(arguments))
# Note that in this case event argument is given to lambda function and therefore need not to be given to "funcToCall"

Mouse events

"<Enter>" # Mouse enters any widget (mouse hover)
"<Leave>" # Mouse leaves any widget (mouse leave)

Mouse Click Events

"<Button-1>" # Left clicked on a widget
"<Button-2>" # Middle clicked on a widget (now a days mouse does not have middle button)
"<Button-3>" # Right clicked on a widget

Keyboard events

"<Key>" # Used pressed a key
"<KeyRelease>" # User released the key

To know more events

Hoverable Button and events

Creating a button which changes colors on hover

from tkinter import *
root = Tk()
def ChangeBG(event,color):
event.widget.config(background=color)
# doing event.widget was not necessary
button = Button(root, text="Hoverable Button", bg="red",fg="white")
button.pack()
button.bind("<Enter>",func =lambda e: ChangeBG(e,"grey"))
button.bind("<Leave>",func =lambda e: ChangeBG(e,"red"))
root.mainloop()

Insertontime and insertofftime options

    CODE
    DESC
# Let us use config to change text widget options
myText.config(insertontime=num1 , insertofftime=num2)
# Default insertontime - 300
# Default insertofftime - 600
insertontime: time for which insertion cursor is on the screen
insertofftime: time for which insertion cursor is not on the screen

Tags in tkinter

# What are Tags
tags are like class in css.
You can set a tag between 2 indices
and then you can configure that tag to change options like background,foreground, font...etc
for that particular region containing that tag
    CODE
    DESC
# Adding a tag
myText.tag_add(tagName ,startIndex,endIndex)
The tagName can be user defined
Few tags are defined by tkinter itself like "sel"
    CODE
    DESC
# Configuring a tag
myText.tag_config(tagName ,**opt)
# **opt includes basic text widget options like font,background,foreground,selectbackground ... etc
# Deleting a tag
myText.tag_delete(tagName)

Marks in text widget

# What are marks
marks are like tag with startIndex=endIndex
They are used to mark a specific index in text widget
# INSERT is also a mark made by tkinter
# Creating a mark
myText.mark_set(markName ,IndexToPlace)
# markName can be user defined
    CODE
    DESC
# Setting up mark gravity
myText.mark_gravity(markName, value)
value can be
LEFT/"left"
or
RIGHT/"right"
RIGHT is default .
